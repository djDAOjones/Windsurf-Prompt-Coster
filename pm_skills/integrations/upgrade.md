---
description: Upgrade an existing project from an older pm-skills version
---

Run the procedure in `pm_skills/prompts/upgrade.md` from step 0.

That file is the canonical source for the upgrade workflow: locate
the latest version, version-check (stop early if already current),
build the work list from `CHANGELOG.md`, classify each path via
`MANIFEST.md`, check for locally-customised framework files,
conditional backup, apply framework changes, merge root templates,
reconcile project memory, readiness check, record, report. Do not
duplicate its steps here.
